# Семейный финансовый Telegram-бот

Бот для учёта долгов, доходов, расходов и финансовых целей. Анализирует ситуацию
и при появлении свободных денег предлагает план распределения с расчётами.
Хранит данные в SQLite, делает резервные копии в Telegram владельцу (по требованию
и автоматически по расписанию).

## Файлы проекта

- `bot.py` — весь код бота (Python 3.11+).
- `requirements.txt` — зависимости (`python-telegram-bot[job-queue]==21.6`).
- `Procfile` — стартовая команда для Railway: `worker: python bot.py`.
- `.gitignore` — исключения для Git (в т.ч. локальная БД и `.env`).
- `README.md` — этот файл.

## Команды бота

```
/start              — приветствие
/help               — список команд

/add_income source=Зарплата amount=120000 monthly=1
/add_expense category=Аренда amount=40000 monthly=1
/add_debt creditor=Sberbank principal=500000 rate=18 payment=15000 term=48 notes="ипотека"
/add_goal name="Подушка безопасности" target=300000 saved=50000 deadline=2026-12-31

/list_debts         — список долгов
/del_debt <id>      — удалить долг
/balance            — сводка месяца
/analyze            — анализ ситуации и рекомендации
/distribute 50000   — куда направить свободные 50000

/myid               — узнать свой Telegram ID (нужен для настройки бэкапов)
/backup             — прислать копию БД владельцу (только OWNER_TELEGRAM_ID)
```

Значения с пробелами — в кавычках: `notes="кредит на машину"`.

## Переменные окружения

| Переменная             | Обязательна | Назначение                                                |
|------------------------|-------------|-----------------------------------------------------------|
| `TELEGRAM_BOT_TOKEN`   | да          | Токен бота от @BotFather.                                 |
| `DB_PATH`              | нет         | Путь к файлу БД. На Railway — `/data/family_finance.db`.  |
| `OWNER_TELEGRAM_ID`    | для бэкапов | Ваш Telegram ID (узнать командой `/myid`).                |
| `BACKUP_INTERVAL_DAYS` | нет         | Период автобэкапа в днях. По умолчанию `7`.               |

## Резервные копии (важно)

Бэкап нужен на случай удаления Railway-проекта, потери доступа к аккаунту или
ошибки в настройках Volume.

- **По требованию.** В Telegram отправляется команда `/backup` владельцу — бот
  присылает файл `family_finance_backup_YYYY-MM-DD_HHMMSS.db` в личку.
- **Автоматически.** Раз в `BACKUP_INTERVAL_DAYS` дней (по умолчанию 7) бот сам
  присылает свежую копию владельцу. Запускается через встроенный
  [`JobQueue`](https://docs.python-telegram-bot.org/en/v21.6/telegram.ext.jobqueue.html).
- **Безопасное копирование.** Используется официальный
  [`sqlite3.Connection.backup`](https://docs.python.org/3/library/sqlite3.html#sqlite3.Connection.backup) —
  он корректно копирует БД, даже если в неё в этот момент идут записи.

Чтобы бэкапы работали, должны быть заданы:
1. `OWNER_TELEGRAM_ID` (узнайте через `/myid`).
2. Зависимость с extra `[job-queue]` (это уже в `requirements.txt`).

`/backup` доступна только пользователю с `OWNER_TELEGRAM_ID`. Полученный `.db`
открывается любой программой для SQLite (например, [DB Browser for SQLite](https://sqlitebrowser.org/)).

## Как развернуть на Railway через GitHub — пошагово

### Шаг 1. Создать бота в Telegram

1. В Telegram открыть [@BotFather](https://t.me/BotFather).
2. Команда `/newbot` → задать имя и username.
3. BotFather пришлёт токен вида `123456789:AAEx...`. Скопировать. Никому не показывать.

Источник: <https://core.telegram.org/bots/features#botfather>.

### Шаг 2. Узнать свой Telegram user_id (для бэкапов)

Это можно сделать в самом боте позже (команда `/myid`), но удобнее заранее.
Самый простой способ — запустить любой публичный бот-«узнай свой ID», например
[@userinfobot](https://t.me/userinfobot) — он пришлёт ваш числовой ID.
(Я не могу гарантировать, что конкретный сторонний бот будет работать в момент
прочтения; альтернатива — установить наш бот без бэкапов, дёрнуть `/myid`,
затем добавить переменную и сделать Redeploy.)

### Шаг 3. Загрузить код на GitHub

1. Создать аккаунт <https://github.com>, если его ещё нет.
2. Создать новый репозиторий, например `family-finance-bot`. Можно сделать его **приватным**.
3. Залить туда файлы: `bot.py`, `requirements.txt`, `Procfile`, `.gitignore`, `README.md`.
   Самый простой путь без терминала — через кнопку «Add file → Upload files» в веб-интерфейсе GitHub.
   Файл `.env` и `family_finance.db` загружать не надо (они в `.gitignore`).

### Шаг 4. Создать проект в Railway

1. Открыть <https://railway.com>, войти через GitHub.
2. New Project → **Deploy from GitHub repo** → выбрать репозиторий
   `family-finance-bot`.
3. Railway соберёт образ через Nixpacks автоматически (Python определится по
   `requirements.txt`). Документация: <https://docs.railway.com/reference/nixpacks>.

### Шаг 5. Добавить переменные окружения

В сервисе Railway → вкладка **Variables** → New Variable, добавить:

```
TELEGRAM_BOT_TOKEN = <токен от BotFather>
DB_PATH            = /data/family_finance.db
OWNER_TELEGRAM_ID  = <ваш числовой Telegram ID>
BACKUP_INTERVAL_DAYS = 7
```

### Шаг 6. Подключить Volume (постоянное хранилище)

Это ключевой шаг для «история всегда сохраняется».

1. В сервисе → **Settings** → **Volumes** → New Volume.
2. Mount path = `/data`.
3. Размер — 1 ГБ хватит на годы текстового учёта.

Документация: <https://docs.railway.com/reference/volumes>.

### Шаг 7. Стартовая команда

Уже задана в `Procfile` (`worker: python bot.py`). Если Railway по какой-то
причине его не подхватит, можно вручную: Settings → Deploy → **Custom Start
Command** = `python bot.py`. Документация: <https://docs.railway.com/guides/start-command>.

### Шаг 8. Deploy и проверка

1. Нажать Deploy. Дождаться зелёного статуса.
2. Открыть вкладку **Deploy Logs** — должна быть строчка `Бот запущен.` и
   `Автобэкап включён: раз в 7 дн., получатель=<ваш id>`.
3. В Telegram открыть свой бот → `/start` → внести пробные данные → `/balance`.
4. Перезапустить сервис в Railway (Restart) и снова сделать `/balance` —
   данные должны остаться.

### Шаг 9. Проверить бэкап

В Telegram отправить боту `/backup`. Бот должен прислать файл `.db` вам в
личку. Это и есть ваша внешняя копия — файл можно скачать на телефон/комп
и хранить где удобно.

## Финансовая методология (источники)

- Метод лавины vs. снежного кома — Consumer Financial Protection Bureau:
  <https://www.consumerfinance.gov/about-us/blog/which-debt-pay-first/>.
- Резерв 3–6 мес. обязательных трат — CFPB:
  <https://www.consumerfinance.gov/an-essential-guide-to-building-an-emergency-fund/>.
  В боте взята нижняя граница (3 месяца).

Оценка экономии в `/distribute` — упрощённая (`сумма × ставка`), верхняя за год.
Точная выгода зависит от типа платежа (аннуитет/дифф.) и оставшегося срока.

## Использование в семейном групповом чате

Бот работает по модели «один чат — один бюджет».

- **В личке с ботом** — это ваш личный бюджет.
- **В семейной группе** (вы + муж + бот) — общий семейный бюджет.
  Все участники видят и пополняют одни и те же доходы, расходы, долги, цели.
- Это два независимых набора данных: записанное в группе не появляется
  в личке и наоборот.

### Как создать общий чат с ботом

1. В Telegram — «Создать группу» → добавить мужа и сам бот по username.
2. Любой участник пишет `/start` — бот ответит и инициализирует общий бюджет.
3. Дальше любые команды (`/add_income`, `/add_expense`, `/balance`, `/analyze`,
   `/distribute`) работают на общем бюджете этого чата.
4. Команда `/backup` доступна только пользователю с `OWNER_TELEGRAM_ID` —
   независимо от того, где она вызвана. Бэкап-файл всегда уходит этому
   пользователю в личку.

### Что важно знать про privacy mode

По умолчанию у бота включён privacy mode: в группе бот видит только команды
(`/...`), ответы на свои сообщения и упоминания @username. Обычные сообщения
семьи он не читает — это документировано Telegram:
<https://core.telegram.org/bots/features#privacy-mode>. Менять этот режим не
надо: всё нужное — команды.

### Узнать chat_id группы

В нужном чате выполнить `/myid`. Бот покажет:
- ваш `user_id` (для бэкапов),
- `chat_id` текущего чата (для понимания, какой бюджет редактируется).

В личке `chat_id == user_id`. В группе `chat_id` — отрицательное число.

## Что важно понимать

- Бот — инструмент учёта и подсказок, не финансовый советник. Все решения
  принимаете вы.
- Бот не подключается к банкам и не списывает платежи.
- Все данные — в вашей БД. На Railway они защищены настолько, насколько
  защищён ваш аккаунт; используйте 2FA.
- В групповом чате доступ к бюджету = доступ к группе. Не добавляйте в эту
  группу посторонних: любой участник сможет видеть и менять записи.
